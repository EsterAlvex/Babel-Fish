Algoritmo/ Algorithm

Um conjunto de regras ou instruções bem definidas para resolver um problema específico.

Um algoritmo é uma sequência específica de passos que são seguidos para resolver um problema ou realizar uma tarefa. Cada passo do algoritmo é uma instrução clara e precisa, e a sequência completa de passos é projetada para resolver o problema de maneira eficiente e correta. Os algoritmos são fundamentais para a ciência da computação e a programação. Eles são usados para criar programas de computador, que instruem os computadores a realizar tarefas específicas. Além disso, os algoritmos também são usados em muitas outras áreas, como matemática, física e engenharia. A palavra "algoritmo" vem do nome do matemático persa do século IX, Al-Khwarizmi, que fez contribuições significativas para a matemática, astronomia, geografia e cartografia. O conceito de algoritmo é uma homenagem a ele. Em resumo, um algoritmo é uma ferramenta poderosa usada para resolver problemas e realizar tarefas de maneira sistemática e eficiente. É uma parte essencial da ciência da computação e da tecnologia moderna.

Array/Matriz

Uma estrutura de dados que armazena uma coleção de elementos (valores ou variáveis), cada um identificado por pelo menos um índice ou chave.

Uma matriz, também conhecida como array, é uma estrutura de dados que armazena uma coleção de elementos, que podem ser de qualquer tipo, como números, strings, objetos, e até mesmo outros arrays. Cada elemento na matriz pode ser acessado por um índice, que é um número que representa a posição do elemento na matriz. As matrizes podem ter uma ou mais dimensões. Uma matriz unidimensional, também conhecida como vetor, tem apenas uma linha ou uma coluna de dados. Uma matriz bidimensional, por outro lado, tem linhas e colunas, semelhante a uma tabela. Matrizes com mais de duas dimensões são chamadas de matrizes multidimensionais. As matrizes são usadas em programação para armazenar e manipular grandes quantidades de dados de maneira eficiente. Elas são fundamentais para muitos algoritmos e estruturas de dados, e são usadas em uma ampla variedade de aplicações, desde a resolução de problemas matemáticos complexos até a criação de gráficos e interfaces de usuário interativas.

Binário/ Binary

Um sistema numérico usado em ciência da computação que consiste em apenas dois números, 0 e 1.

O código binário é uma forma de representar informações usando apenas dois dígitos, geralmente 0 e 11. Esta linguagem é a base de todos os computadores e sistemas digitais modernos. A história do código binário começa com o filósofo e matemático Gottfried Wilhelm Leibniz, que, no século XVII, concebeu o sistema numérico binário. Leibniz foi inspirado pelo I Ching, um antigo texto chinês, que empregava um sistema de símbolos dualísticos para representar o universo. O sistema binário ganhou mais destaque durante o século XIX, com os trabalhos de George Boole. Boole desenvolveu a álgebra booleana, que emprega operações lógicas sobre valores binários e é a base para os circuitos digitais modernos. No século XX, o matemático e cientista da computação Claude Shannon aplicou a álgebra booleana ao projeto de circuitos digitais, lançando as bases para a computação digital moderna. Os conceitos fundamentais do código binário incluem o bit, a unidade básica de informação, e o byte, que é composto por 8 bits. No contexto do código binário, cada bit representa um valor de 0 ou 1.

Booleano/ Boolean

Um tipo de dado que tem um de dois possíveis valores (geralmente denotado verdadeiro e falso) destinado a representar os dois valores de verdade da lógica e da álgebra booleana.

A álgebra booleana foi criada pelo matemático George Boole (1815-1864) como uma tentativa de tornar precisas as regras da lógica. No século XX, entretanto, ele encontrou usos incríveis em campos como a computação digital. George Boole criou um sistema matemático para lógica. Na álgebra booleana, existem apenas dois valores possíveis: Verdadeiro (1) ou Falso (0). As operações na álgebra booleana não são exatamente como a adição, subtração, multiplicação ou divisão que estamos acostumados a ver na álgebra. Em vez disso, usamos AND, OR, NOT e operações semelhantes. Qualquer variável booleana, p , q , r , etc., pode assumir o valor 1 ou 0. Esse valor é chamado de valor verdadeiro (1 significa Verdadeiro e 0 significa Falso). As letras podem representar declarações específicas. Uma tabela verdade pode ser usada para definir cada operação. É também um método para simplificar circuitos lógicos (ou às vezes chamados de circuitos de comutação lógica) em eletrônica digital.

Bug/ Bug

Um erro, falha ou defeito no design, desenvolvimento ou operação do software que faz com que ele produza um resultado incorreto ou inesperado, ou se comporte de maneiras não intencionais.

"Bug" é um termo usado na informática para se referir a falhas inesperadas que ocorrem ao executar algum software ou usar um hardware. Esses erros imprevisíveis podem prejudicar o funcionamento correto de uma tecnologia e podem desencadear problemas incômodos, como travamentos e roubo de informações sigilosas. Alguns bugs são mais graves e necessitam de reparação rápida, pois podem colocar em risco o sistema de segurança de um aparelho e, consequentemente, comprometer dados como senhas e contas bancárias. A palavra inglesa "bug" significa literalmente "inseto". A origem do termo para designar falhas em softwares e hardwares é atribuída a um erro que ocorreu em 9 de setembro de 1947, no computador Mark II, operado pela Marinha dos Estados Unidos. O operador William Burke encontrou dentro da máquina uma mariposa que estava presa entre os fios, causando uma falha no funcionamento do computador. Antes de desenvolvedores lançarem um programa ou sistema de hardware, vários testes são realizados para evitar os bugs e quaisquer casos de prejuízos ou riscos à segurança por mau funcionamento. Quando um bug é descoberto, é preciso descobrir a origem da falha inesperada e oferecer rapidamente atualizações gratuitas aos usuários do hardware ou software para resolver o problema.

"Classe/ Class"

Em programação orientada a objetos, uma classe é um modelo extensível de código de programa para criar objetos, fornecendo valores iniciais para o estado (variáveis membro) e implementações de comportamento (funções membro ou métodos).

Na programação, uma "classe" é uma estrutura abstrata que contém funções e variáveis. As funções dentro da classe definem seu comportamento e sua forma de utilização. As variáveis armazenam determinados valores durante o tempo de execução. O conjunto das funções e variáveis dentro de uma classe é chamado de "encapsulamento". Em outras palavras, uma classe é um tipo de dados definido pelo programador. Ela pode incluir enumerações (enum), uniões (union), estruturas (struct) e classes em sentido estrito (class). Além de dados (membros-dados), uma classe pode também conter funções de manipulação desses dados (membros-funções), restrições de acesso a ambos (dados e funções) e redefinições de quase todos os operadores de C++ para objetos da classe. Um objeto é uma instância de uma classe. Em vez de variáveis (do tipo T), fala-se em objetos (da classe ou tipo T). Um objeto tem identidade, estado e comportamento. A identidade é representada pelo endereço do objeto, o estado é representado pelos valores dos membros-dados (também chamados atributos em outras linguagens), e o comportamento é descrito pelos membros-função (também chamados métodos em outras linguagens), incluindo funções que definem operadores. As classes são fundamentais na programação orientada a objetos, pois permitem a criação de estruturas complexas e personalizadas que podem ser usadas para modelar conceitos do mundo real dentro de um programa.

"Código/ Code"

Descreve o texto escrito usando uma linguagem de programação por um programador de computador.

O código é uma série de instruções escritas em uma linguagem de programação específica. Essas instruções são executadas por um computador para realizar uma tarefa específica. Existem muitas linguagens de programação diferentes, cada uma com suas próprias regras (sintaxe) e usos. Algumas das linguagens de programação mais populares incluem Python, Java, JavaScript, C++, entre outras.

"Compilador/ Compiler"

Um programa de computador que traduz o código do computador escrito em uma linguagem de programação (a linguagem fonte) para outra linguagem (a linguagem alvo)

Um compilador é um programa de computador que traduz o código-fonte escrito em uma linguagem de programação para uma linguagem de máquina. Isso permite que o código seja executado por um computador. O compilador lê e analisa o código-fonte da aplicação, ou seja, o código que nós escrevemos, e gera a partir dele um código binário que pode ser executado. Ele é um tradutor de linguagens de programação casuais para linguagens de programação do nível da máquina. Existem muitas linguagens de programação diferentes, e cada uma delas tem seu próprio compilador. Por exemplo, o GCC (GNU Compiler Collection) é um compilador muito popular para a linguagem C. Os compiladores realizam várias etapas para traduzir o código-fonte em código de máquina. Isso inclui a análise do código-fonte, a otimização do código e a geração do código de máquina.

"Constante/ Constant "

Uma quantidade que não muda seu valor ao longo do tempo. Uma constante é uma quantidade universal, onde permanece a mesma para todos os aspectos da computação.

Uma constante na programação é um valor fixo que um programa não pode modificar durante sua execução. Esses valores fixos também são conhecidos como literais. Uma constante é tratada como uma variável comum, exceto que seu valor não pode ser alterado depois de definido. As constantes podem ser de qualquer tipo de dados, como, por exemplo, um caractere ou um número. As constantes são usadas na programação para armazenar valores fixos. Eles são de interesse para definir valores que serão usados várias vezes em um programa

"Construtor/ Constructor"

Um método especial da classe que é automaticamente invocado sempre que uma instância da classe é criada.

Um construtor é um método especial em uma classe de programação orientada a objetos. Ele é chamado automaticamente quando um objeto da classe é criado. O nome do construtor é o mesmo que o nome da classe e ele não tem um tipo de retorno. O propósito principal de um construtor é inicializar um objeto recém-criado para um estado inicial seguro antes que seja usado. Ele pode ser usado para definir valores padrão, limitar a instanciação e escrever códigos flexíveis e fáceis de ler.

"Estrutura de Dados/ Date Structure "

É uma maneira particular de armazenar e organizar dados em um computador para que possam ser usados de maneira eficiente. Exemplos de estruturas de dados incluem arrays, pilhas, filas, listas vinculadas e árvores.

Estruturas de Dados são uma maneira de organizar e armazenar dados para que possam ser acessados e trabalhados de maneira eficiente. Elas definem a relação entre os dados, e as operações que podem ser executadas sobre eles. Existem vários tipos de estruturas de dados, cada uma com suas próprias forças e fraquezas. A escolha da estrutura de dados correta para uma tarefa específica depende da natureza dos dados e do que você precisa fazer com eles. Por exemplo, se você precisa acessar elementos em uma ordem específica, uma lista encadeada pode ser a melhor escolha. Se você precisa de acesso rápido a elementos com base em uma chave, uma tabela hash pode ser a melhor escolha. A compreensão das estruturas de dados e de quando usá-las é uma parte fundamental da ciência da computação e da programação.

"Banco de Dados/ Database"

É uma coleção organizada de dados. Esses dados podem ser criados, atualizados, gerenciados e armazenados de maneira sistemática para facilitar o acesso e a análise.

A estrutura de um banco de dados é a forma como os dados são organizados e armazenados. Ela é projetada para facilitar o gerenciamento, acesso, recuperação e atualização de dados de maneira eficiente. A escolha do modelo de banco de dados depende das necessidades específicas do sistema que está sendo desenvolvido. Por exemplo, para aplicações que requerem alta performance e escalabilidade, um banco de dados NoSQL pode ser a melhor escolha. Por outro lado, para aplicações que requerem uma forte consistência de dados e integridade referencial, um banco de dados relacional pode ser mais adequado. A compreensão das estruturas de banco de dados e de quando usá-las é uma parte fundamental da ciência da computação e da programação.

"Depuração/ Debugging"

É o processo de encontrar e corrigir bugs (defeitos ou problemas que impedem a operação correta) dentro de programas de computador, software ou sistemas.

A depuração, ou debugging, é um processo essencial no desenvolvimento de software que envolve a identificação e correção de erros, também conhecidos como bugs, em um programa de computador. Durante a depuração, os desenvolvedores usam ferramentas chamadas depuradores para executar o programa passo a passo, para que possam rastrear e corrigir os erros. Os depuradores permitem que os desenvolvedores parem a execução do programa em pontos específicos chamados pontos de interrupção e examinem os valores das variáveis. A depuração é uma habilidade essencial para qualquer desenvolvedor e é uma parte integral do ciclo de vida do desenvolvimento de software. Ela ajuda a garantir que o programa funcione corretamente e atenda aos requisitos especificados. Além disso, a depuração também pode envolver a revisão do código para entender o fluxo de execução, a verificação de logs para identificar eventos anômalos, a realização de testes para gerar erros e a investigação do estado do sistema para rastrear as causas dos erros. Em resumo, a depuração é um processo sistemático para detectar, diagnosticar e corrigir os erros em um programa de computador, tornando-o mais eficiente, eficaz e livre de erros.

"Exceção/ Exception "

É um evento que ocorre durante a execução de um programa, que interrompe o fluxo normal das instruções do programa. Quando ocorre um erro dentro de um método, o método cria um objeto e o entrega ao sistema de tempo de execução.

Exceções, ou exceptions, são eventos que ocorrem durante a execução de programas que interrompem o fluxo normal de instruções. Em outras palavras, uma exceção é uma condição anormal que surge em um bloco de código. Quando uma exceção é acionada, o controle normal do programa é interrompido e o programa/software passa a procurar por algo que possa manipular a exceção. Isso é feito através de um processo chamado "captura de exceção" ou "tratamento de exceção". O tratamento de exceções é muito importante e útil em programação. Em muitas linguagens de programação modernas, as exceções são implementadas de maneira semelhante. Elas são divididas em dois tipos principais: exceções verificadas (que requerem tratamento explícito pelo programador) e exceções não verificadas (que não precisam ser tratadas explicitamente). Em resumo, as exceções são fundamentais para a construção de software robusto e resistente a erros, pois fornecem uma maneira estruturada de detectar, sinalizar e lidar com erros durante a execução do programa.

"Função/ Function"

É uma unidade de código que realiza uma tarefa. Pode ser chamada e reutilizada várias vezes. Você pode passar informações para uma função e ela pode enviar informações de volta.

Na programação, uma "função" é um bloco de código nomeado que realiza uma tarefa específica. Funções são fundamentais para a estruturação do código e para evitar a repetição. Elas permitem que os programadores reutilizem o mesmo código em diferentes partes de um programa sem ter que copiá-lo e colá-lo várias vezes. Uma função geralmente toma alguns valores de entrada, chamados de "parâmetros", executa operações específicas e retorna um resultado. O uso de funções torna o código mais legível e fácil de manter, pois cada função é projetada para realizar uma tarefa específica. Além disso, as funções promovem a reutilização de código e a modularidade. As funções são uma parte essencial de todas as linguagens de programação e são fundamentais para a construção de programas eficientes e manuteníveis.

"Herança/ Inheritance"

Na programação orientada a objetos, a herança é um mecanismo que permite que uma classe derive propriedades e comportamentos de outra classe.

A "herança" é um princípio fundamental da programação orientada a objetos (OOP) que permite criar uma nova classe a partir de uma classe existente. A nova classe, conhecida como "classe derivada" ou "subclasse", herda os atributos e métodos da "classe base" ou "superclasse". A herança permite o "reuso de código", pois você pode criar uma nova classe que incorpora atributos e comportamentos já definidos em outra classe, e adicionar ou sobrescrever apenas o que precisa ser diferente. Isso leva a um código mais organizado e manutenível. Além disso, a herança permite a criação de hierarquias de classes que representam relações do tipo "é um". Por exemplo, se temos uma classe "Animal" com atributos como "nome" e "idade" e métodos como "comer" e "dormir", podemos criar uma classe "Cachorro" que herda de "Animal" e adiciona atributos específicos como "raça" e métodos como "latir". É importante notar que a herança deve ser usada com cuidado, pois nem todas as relações do mundo real são melhor representadas como relações de herança. Em muitos casos, a composição (onde uma classe contém outra) pode ser uma escolha de design mais apropriada.

"Interface/ Interface "

Em termos simples, uma interface é um contrato. Este contrato declara o comportamento de algum componente. Ele define a interação entre os componentes que usam a interface. Ele também define a própria interface.

Na programação orientada a objetos, uma "interface" é uma definição de um contrato que outras classes podem implementar. Em outras palavras, uma interface é um grupo de métodos relacionados com corpos vazios. Uma interface define um conjunto de métodos que uma classe deve implementar. Isso permite que objetos sejam manipulados abstratamente como instâncias de uma interface, em vez de instâncias de uma classe específica. Isso é útil quando várias classes podem realizar a mesma ação, mas de maneiras diferentes. Por exemplo, você pode ter uma interface chamada "Forma" com um método "desenhar". Então, você pode ter várias classes como "Círculo", "Retângulo" e "Triângulo" que implementam a interface "Forma" e fornecem sua própria implementação do método "desenhar". A interface ajuda a garantir a segurança do tipo e a consistência em seu código, permitindo que você use polimorfismo para escrever código que pode trabalhar com qualquer classe que implemente uma determinada interface. Isso torna seu código mais flexível e extensível, pois novas classes podem ser adicionadas sem alterar o código existente, desde que implementem a interface correta.

"Laço de Repetição/ Loop "

É uma estrutura de programação que repete uma sequência de instruções até que uma condição especificada seja atingida.

Na programação, um laço de repetição ou loop é uma estrutura de controle que permite executar um bloco de código várias vezes. Existem vários tipos de loops, como for, while e do-while, dependendo da linguagem de programação. O loop for é geralmente usado quando sabemos o número exato de vezes que o bloco de código deve ser executado. O loop while é usado quando queremos que o bloco de código seja executado até que uma determinada condição seja falsa.

"Método/ Method"

São funções anexadas a classes específicas (ou instâncias) na programação orientada a objetos.

Um método é uma função que é associada a um objeto. Em outras palavras, um método é uma função que é definida dentro de uma classe na programação orientada a objetos. Os métodos permitem que os objetos tenham comportamentos e capacidades, e são usados para acessar e modificar os dados de um objeto. Cada método tem um nome e pode receber vários parâmetros. Quando um método é chamado, ele realiza uma ação específica que é definida no corpo do método. Os métodos são fundamentais para a encapsulação e a abstração, dois conceitos chave na programação orientada a objetos.

"Objeto/ Object "

Na programação orientada a objetos (OOP), um objeto é um tipo de dado abstrato criado por um desenvolvedor. Pode incluir várias propriedades e métodos e pode até conter outros objetos.

Na programação, um "objeto" é uma entidade que possui "estado", "comportamento" e "identidade". Os objetos são a base da "programação orientada a objetos (POO)", um paradigma de programação que usa objetos e suas interações para projetar aplicações e programas de software. A POO oferece benefícios como modularidade, reutilização de código e flexibilidade, tornando-a uma escolha popular para muitos desenvolvedores.

"Operador/ operator "

Em programação, um operador é um símbolo que realiza operações específicas em um ou mais operandos (variáveis). Os operadores são usados em muitos programas comuns e podem manipular dados individuais e elementos dentro de matrizes e objetos.

É um símbolo que realiza uma operação específica em um ou mais operandos. Os operandos podem ser variáveis, valores ou expressões. Os operadores são classificados em diferentes tipos, dependendo do número de operandos que eles trabalham e do tipo de operação que realizam. Alguns dos tipos mais comuns de operadores incluem: "Operadores aritméticos" que realizam operações matemáticas básicas como adição (+), subtração (-), multiplicação (*), divisão (/) e módulo (%). "Operadores de comparação" que comparam dois operandos e retornam um valor booleano (verdadeiro ou falso). Exemplos incluem igual a (==), não igual a (!=), maior que (>), menor que (<), maior ou igual a (>=), e menor ou igual a (<=). "Operadores lógicos" que realizam operações lógicas como AND (&&), OR (||) e NOT (!). "Operadores de atribuição" que atribuem um valor a uma variável. O operador de atribuição mais comum é o sinal de igual (=), mas existem outros, como +=, -=, *=, e /=, que combinam uma operação aritmética com uma atribuição. Os operadores são fundamentais para a construção de expressões e instruções em muitas linguagens de programação.

"Parâmetro/ Parameter"

Um parâmetro é uma variável especial usada em uma função para se referir a um dos dados fornecidos como entrada para a função. Esses dados são os valores dos argumentos com os quais a função será chamada.

Parâmetro é um termo usado em programação para se referir a uma variável especial usada em uma função para receber um valor ou vários valores quando a função é chamada. Em outras palavras, os parâmetros são os valores de entrada que uma função usa para realizar seus cálculos. Por exemplo, em uma função que adiciona dois números, os dois números seriam os parâmetros. Os parâmetros permitem que as funções sejam mais flexíveis e reutilizáveis, pois não precisam ter valores fixos codificados dentro delas.

"Recursão/ Recursion"

Recursão é um processo que chama a si mesmo direta ou indiretamente e a função correspondente é chamada de função recursiva. A recursão tem algumas propriedades importantes, como a capacidade de resolver um problema dividindo-o em subproblemas menores.

Recursão é um conceito em ciência da computação onde uma função chama a si mesma em sua definição. Uma função recursiva resolve um problema dividindo-o em subproblemas menores que são do mesmo tipo que o problema original. A recursão continua até que seja atingido um caso base, que é um caso do problema que pode ser resolvido diretamente. A recursão é uma ferramenta poderosa em programação, mas deve ser usada com cuidado, pois pode levar a um loop infinito se o caso base não for definido corretamente ou se a função recursiva não estiver se aproximando do caso base em cada recursão. Além disso, a recursão pode consumir muita memória se o número de chamadas recursivas for muito grande, pois cada chamada à função requer espaço na pilha de chamadas.

"Referência/ Reference"

Em programação, uma referência é um valor que permite a um programa acessar indiretamente um determinado dado, como o valor de uma variável ou um registro, na memória do computador ou em algum outro dispositivo de armazenamento.

Referência em programação é um valor que permite manipular um objeto, variável ou função específica diretamente. Uma referência é como um ponteiro para um local na memória onde o objeto, variável ou função reside. Isso permite que você acesse e modifique o valor do objeto, variável ou função sem ter que copiá-lo. Referências são particularmente úteis quando você está trabalhando com objetos grandes, onde copiar o objeto inteiro seria ineficiente.

"Sintaxe/ Syntax"

A sintaxe de uma linguagem de programação são as regras que definem as combinações de símbolos que são consideradas declarações ou expressões corretamente estruturadas nessa linguagem.

Sintaxe refere-se ao conjunto de regras que definem como programas em uma linguagem de programação específica são estruturados. Cada linguagem de programação tem sua própria sintaxe que determina como os elementos da linguagem (como variáveis, funções, loops, etc.) são definidos e organizados. Erros de sintaxe, também conhecidos como erros de compilação, ocorrem quando o código não segue a sintaxe correta e, como resultado, o compilador ou interpretador não pode entender o código. Aprender a sintaxe correta é uma parte fundamental do aprendizado de uma nova linguagem de programação.

"Variável/ Variable"

Em ciência da computação, uma variável é um local de armazenamento abstrato emparelhado com um nome simbólico associado, que contém alguma quantidade conhecida ou desconhecida de dados ou objeto referido como valor.

Uma variável na programação é um nome simbólico para um espaço de memória que o programa pode manipular. A variável contém um valor conhecido ou desconhecido, referido como o valor da variável. As variáveis são usadas para armazenar informações que serão manipuladas ou usadas pelo programa. Por exemplo, uma variável pode conter dados como um número, uma string, uma lista, etc.

"Módulo/ Module"

A programação modular é uma técnica de design de software que enfatiza a separação da funcionalidade de um programa em módulos independentes e intercambiáveis, cada um contendo tudo o necessário para executar apenas um aspecto da funcionalidade desejada.

Um módulo, por outro lado, é um arquivo contendo definições e instruções de Python. O arquivo contém definições de funções, classes e variáveis que podem ser usadas em outros programas Python. Portanto, um módulo é uma maneira de organizar código relacionado em um único arquivo, tornando o código mais legível e reutilizável. Os módulos podem ser importados para outros programas Python, permitindo o uso das funções e variáveis definidas no módulo.

"Pacote/ Package"

Um pacote é um namespace que organiza um conjunto de classes e interfaces relacionadas. Conceitualmente, você pode pensar em pacotes como sendo semelhantes a diferentes pastas no seu computador.

Na programação é uma maneira de agrupar código relacionado em um único local. Ele contém um conjunto de funções, classes ou ambos, que podem ser importados e utilizados em outros programas. Os pacotes ajudam a organizar o código de maneira lógica e a evitar conflitos de nomes entre diferentes partes do código. Eles são uma maneira eficaz de gerenciar e organizar grandes projetos de software. Além disso, os pacotes podem ser facilmente compartilhados e reutilizados, o que promove a reutilização de código e a modularidade.

"Biblioteca/ Library"

Em ciência da computação, uma biblioteca é uma coleção de recursos não voláteis usados por programas de computador, muitas vezes para desenvolvimento de software.

Na programação é uma coleção de implementações de comportamentos, escritas em termos de uma linguagem de programação, que tem uma interface bem definida por qual o comportamento é invocado. Em outras palavras, é um conjunto de funções que são usadas em um programa para evitar escrever o mesmo código várias vezes. As bibliotecas permitem que você use o código escrito por outras pessoas para realizar tarefas comuns, economizando tempo e esforço. Elas são uma parte essencial da maioria dos programas modernos, pois fornecem código que é testado, otimizado e eficiente.

"Framework/ Framework "

Um framework é um conjunto predefinido de soluções que resolve problemas comuns de desenvolvimento.

É uma estrutura ou plataforma que você usa como base para desenvolver seu próprio aplicativo. Ele fornece um conjunto padrão de estruturas (classes e funções) que pode ser usado como base para criar novos programas. Um framework geralmente inclui código que define estruturas de controle de fluxo básicas e define a organização geral de um sistema de software. Ele ajuda a padronizar o processo de desenvolvimento, fornecendo um conjunto comum de práticas e permitindo que os desenvolvedores se concentrem na lógica específica do aplicativo em vez de nos detalhes de baixo nível. Além disso, os frameworks geralmente vêm com suporte para a construção e implantação de aplicativos, o que pode simplificar muito o processo de desenvolvimento.
